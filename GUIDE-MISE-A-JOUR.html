3. Navigation et footer (menu)

Le menu et le footer sont gérés depuis un seul endroit. Plus besoin de modifier 6 fichiers — tu modifies un seul fichier, tu lances le script, et toutes les pages sont mises à jour.

Comment ça marche

Le menu vit dans _includes/nav.html et le footer dans _includes/footer.html. Ces deux fichiers sont la source de vérité unique. Le script build.js injecte leur contenu dans les 6 pages HTML du site.

Pour modifier le menu ou le footer

  1. Ouvre _includes/nav.html (pour le menu) ou _includes/footer.html (pour le footer)
  2. Fais ta modification
  3. Lance la commande : node build.js
  4. C'est fait — les 6 pages sont mises à jour automatiquement
# Depuis le dossier du projet :
node build.js

# Résultat :
#   OK    index.html (2 bloc(s) mis à jour)
#   OK    formation.html (2 bloc(s) mis à jour)
#   OK    ecstatic-dance.html (2 bloc(s) mis à jour)
#   OK    a-propos.html (2 bloc(s) mis à jour)
#   OK    partenaires.html (2 bloc(s) mis à jour)
#   OK    contact.html (2 bloc(s) mis à jour)
Ne modifie JAMAIS le menu ou le footer directement dans les fichiers HTML des pages (index.html, formation.html, etc.). Tes changements seraient écrasés au prochain node build.js. Modifie toujours dans _includes/.

Structure du menu (_includes/nav.html)

Le fichier contient deux parties :

<!-- 1. Menu desktop -->
<nav class="nav" id="nav">
  <ul class="nav-links">
    <li><a href="formation.html">Formation</a></li>
    <li><a href="ecstatic-dance.html">Ecstatic Dance</a></li>
    ...
  </ul>
</nav>

<!-- 2. Menu mobile (juste en dessous) -->
<div class="nav-mobile" id="mobileMenu">
  <a href="formation.html" class="nav-mobile-link">Formation</a>
  ...
</div>
Pense à modifier les deux parties (desktop + mobile) quand tu ajoutes ou renommes un lien. Elles sont dans le même fichier _includes/nav.html.

Éléments du footer (_includes/footer.html)

  • Liens de navigation : même liste que le menu principal
  • Liens réseaux sociaux : Instagram, Facebook, YouTube, SoundCloud, Mixcloud
  • Espace élèves : lien vers https://membres.benjaminbossu.com
  • Année copyright : © 2026 DJing Is A Medicine
  • Mentions légales & Confidentialité : lien vers mentions-legales.html (page unique)

Comment le build.js fonctionne (pour info)

Dans chaque fichier HTML, le menu est entouré de marqueurs :

<!-- NAV:START -->
  ... le menu est ici ...
<!-- NAV:END -->

Le script remplace tout ce qui est entre NAV:START et NAV:END par le contenu de _includes/nav.html. Pareil pour FOOTER:START / FOOTER:END.

4. Modifier les textes

Les textes sont directement dans le HTML. Pour les retrouver facilement, chaque section est précédée d'un commentaire clair :

<!-- ==========================================
   SECTION 2 — PHILOSOPHIE
   ========================================== -->

Structure type d'une section

<section class="philosophy" id="philosophie">
  <div class="container">
    <div class="section-label">Vision</div>          ← petit label au-dessus
    <h2 class="section-title">Titre principal</h2>    ← titre de section
    <p class="section-text">Ton texte ici...</p>      ← paragraphe
  </div>
</section>
Pour modifier un texte : ouvre le fichier HTML, cherche la phrase avec Ctrl+F, modifie-la, enregistre. C'est tout. Pas besoin de lancer build.js pour les textes — seulement pour le menu et le footer.

Pour remplacer une image

  1. Place ta nouvelle image dans le bon dossier
  2. Donne-lui le même nom que l'ancienne (ou modifie le src dans le HTML)
  3. Format recommandé : JPG ou WebP, largeur max ~1200px, qualité 80%

Pour ajouter une image

<img src="images/formation/ma-nouvelle-photo.jpg"
     alt="Description de la photo"
     loading="lazy">
Toujours mettre loading="lazy" sur les images qui ne sont pas dans le hero (ça optimise le chargement).

6. Témoignages

Page d'accueil (index.html)

L'accueil a un système de rotation automatique des témoignages. La liste est dans un bloc <script> en bas de index.html :

const testimonialPool = [
  {
    stars: '★★★★★',
    quote: '"Le texte du témoignage..."',
    initials: 'LC',         // ← initiales pour l'avatar
    name: 'Laura C.',       // ← nom affiché
    role: 'Participante formation'  // ← sous-titre
  },
  // ... autres témoignages
];

Pour ajouter un témoignage (accueil)

Ajoute un nouvel objet dans le tableau testimonialPool :

{
  stars: '★★★★★',
  quote: '"Le texte du nouveau témoignage ici."',
  initials: 'AB',
  name: 'Alice B.',
  role: 'Participante retraite'
},
Attention aux guillemets : dans le JavaScript, les apostrophes dans le texte doivent être échappées avec \'. Exemple : 'j\'ai adoré' et non 'j'ai adoré'.

Il y a aussi les "big quotes" qui tournent :

const bigQuotes = [
  '« Je ne pensais pas être capable… »',
  '« Ça a changé ma vision de la musique… »',
  '« Un avant et un après… »',
];

Page Formation (formation.html)

Les témoignages de la page formation sont directement dans le HTML (pas de rotation JS). Cherche SECTION — TEMOIGNAGES dans le fichier.

Un témoignage type :

9. Formulaire de contact

Il existe deux formulaires sur le site :

PageFormulaireChamps
index.html + formation.htmlCompletPrénom, nom, email, téléphone, ville, niveau DJ, niveau espaces, provenance, sujet, motivation (conditionnel), message, newsletter
contact.htmlSimplifiéPrénom, nom, email, téléphone, sujet, message, newsletter

Où vont les données ?

Les formulaires envoient les données à Systeme.io via un proxy :

  1. D'abord vers /contact.php (sur LWS, serveur PHP)
  2. En fallback, vers /.netlify/functions/contact (sur Netlify)

Le contact est créé ou mis à jour dans Systeme.io, et des tags (formsite, newsletter) lui sont assignés.

Modifier les options du formulaire

Pour changer les options d'un menu déroulant (ex: le sujet) :

<select name="subject" required>
  <option value="" disabled selected>Sujet</option>
  <option value="dj-holistique">Retraite formation DJ holistique</option>
  <option value="danses-48">D.A.N.S.E.S 48</option>
  <!-- Ajoute une option ici si besoin -->
</select>
Important : L'attribut value est ce qui est envoyé à Systeme.io. Le texte visible est entre les balises <option>. Si tu ajoutes une option, assure-toi que le value est en minuscules sans accents.

Le champ "motivation" conditionnel

Quand le sujet "DJ holistique" est sélectionné, un champ supplémentaire apparaît (échelle de 1 à 10). C'est géré par js/contact.js — pas besoin d'y toucher sauf si tu veux modifier l'échelle.

Polices

  • Outfit (Google Fonts) : texte courant, poids 400 et 600
  • Fredoka (Google Fonts) : certains titres, poids 600
  • Decalotype Bold (locale dans fonts/) : titres hero principaux

Alternance des sections

Le site alterne entre deux types de fonds :

  • Transparent / parallax : fond sombre avec le mandala animé visible derrière
  • Opaque (#D5CBC0) : fond beige/crème, texte sombre

Les séparateurs en vague (wave-separator) font la transition entre les deux.

Tu n'as normalement pas besoin de toucher aux fichiers JS. Voici quand même ce que chacun fait :

L'attribut reveal

Tout élément avec la classe reveal apparaîtra en animation quand il entre dans l'écran au scroll. Tu peux ajouter des délais :

<div class="reveal">Apparaît en premier</div>
<div class="reveal reveal-delay-1">Apparaît 0.1s après</div>
<div class="reveal reveal-delay-2">Apparaît 0.2s après</div>

13. Déploiement

Hébergement

  • LWS : hébergement principal de Benjamin (PHP + fichiers statiques)
  • Netlify : maquette de démo (déploiement automatique depuis GitHub)

Workflow de mise à jour

  1. Modifier les fichiers (textes dans le HTML, ou menu/footer dans _includes/)
  2. Si menu/footer modifié : lancer node build.js
  3. Uploader les fichiers sur LWS (FTP) ou push sur GitHub pour Netlify

Commandes git de base

# Voir ce qui a changé
git status

# Ajouter les fichiers modifiés
git add index.html formation.html

# Créer un commit
git commit -m "Mise à jour des textes accueil et formation"

# Envoyer sur le serveur (Netlify se déploie automatiquement)
git push

14. Zones à ne pas toucher

Ne modifie pas ces éléments sauf si tu sais ce que tu fais :
  • Le mandala SVG dans le hero de index.html — géométrie sacrée calculée précisément
  • Les wave-separators — les <svg> avec animations sont calibrés pour les transitions entre sections
  • Les marqueurs <!-- NAV:START --> / <!-- NAV:END --> et <!-- FOOTER:START --> / <!-- FOOTER:END --> — utilisés par build.js
  • build.js — le script de build, ne pas modifier sauf pour ajouter une page
  • js/mandala.js — l'animation Canvas est optimisée (IntersectionObserver, pause hors écran...)
  • js/main.js — les interactions (curseur, tilt 3D, menu) sont connectées entre elles
  • contact.php et netlify/functions/contact.mjs — contiennent la clé API Systeme.io
  • L'ordre des <link> et <script> dans le <head> et fin de <body>
  • Les attributs id — ils sont utilisés par le JavaScript (ex: id="nav", id="contactForm")
  • data-magnetic — attribut qui active l'effet magnétique sur les boutons
  • class="reveal" — ne pas supprimer si tu veux garder les animations au scroll

Checklist rapide de mise à jour

  1. Ouvrir le bon fichier HTML
  2. Ctrl+F pour trouver le texte
  3. Modifier, enregistrer

  2. Formation : dupliquer un bloc .testimonial-card dans le HTML
  3. Ajouter la photo avatar dans images/formation/
  4. Upload FTP ou git push

Je veux ajouter un partenaire

  1. Copier un bloc .partenaire-card dans partenaires.html
  2. Modifier nom, description, lien, image
  3. Photo dans images/partenaires/
  4. Upload FTP ou git push

Je veux modifier le menu ou le footer

  1. Modifier _includes/nav.html (menu) ou _includes/footer.html (footer)
  2. Lancer node build.js
  3. Upload FTP ou git push
